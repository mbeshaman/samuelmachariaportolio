usbasp programmer

Interface between a computer and an AVR microcontroller requires a USB to TTL converter.This is because most AVR microcontrollers have do not have inbuilt USB hardware. However the hardware USB function can be emulated by firmware written into the bootloader section of flash memory thus the bootloader can be used to program the flash memory. A condition can be set to switch between bootloader program and main program. In the above circuit pin PD7 is grounded before resetting the microcontroller.This initiates the bootloader program. The programming is done by AVRDUDE commands. Since the programming port does not display in arduino ide the GUI application below would be necessary. the application can be used with arduino module,standalone chip or even a remote programmer. The application is designed for both Linux and Windows environment.

ESP8266 AVRISP

esp8266avrisp allows programming AVR microcontrollers wirelessly as shown.The port is different from avrisp,for example when using arduino the COM port would be something like COM1 in windows OS or /dev/ttyACM0 in Linux environment in esp8266avrisp the port as shown is net:192.168.4.1 which is the IP address of the esp module.
